# Workplace Development Rules

## Team Collaboration
- Follow established code review process
- Use consistent commit message format
- Document architectural decisions
- Maintain clean git history

## Production Considerations
- Never deploy untested code
- Follow deployment procedures
- Implement proper monitoring
- Have rollback plans ready

## Documentation Standards
- Keep README updated
- Document breaking changes
- Maintain API documentation
- Write clear deployment guides

## Code Review Guidelines
- Focus on logic and security
- Check for performance issues
- Verify test coverage
- Ensure documentation updates

## Emergency Procedures
- Have incident response plan
- Know rollback procedures
- Maintain communication channels
- Document post-mortem findings

## Compliance
- Follow company coding standards
- Adhere to security policies
- Maintain audit trails
- Respect data privacy regulations
